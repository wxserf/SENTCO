# Contributing to Sentience

Thank you for your interest in contributing to Sentience!

## How to Contribute

- **Bug Reports:**  
  Please open a [GitHub Issue](https://github.com/yourname/sentience-eve/issues) with steps to reproduce and relevant logs or screenshots.

- **Feature Requests:**  
  Start a Discussion or open an Issue labeled “enhancement.”

- **Pull Requests:**  
  1. Fork the repo, branch from `main` or `dev`
  2. Write clear, documented code with tests where possible
  3. Describe your PR and reference any relevant Issues
  4. Code review and merge by a maintainer

## Development Guidelines

- Follow PEP8 (Python) and StandardJS (Node)
- Use semantic commit messages (see [Conventional Commits](https://www.conventionalcommits.org/))
- Write docstrings and comments for major logic
- All sensitive API credentials in `.env`, never in code

## Code of Conduct

All contributors must abide by the [Contributor Covenant](https://www.contributor-covenant.org/).

## Questions?

Join our [Discord server](https://discord.gg/example) or open a Discussion!
